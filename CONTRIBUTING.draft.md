# Contributing Guide Draft

Thank you for considering a contribution. This file is a draft and should be reviewed before the repository is made public.

## Ways To Contribute

- Report bugs with clear reproduction steps.
- Suggest small, focused improvements.
- Improve documentation and examples.
- Submit pull requests that solve a specific issue.

## Before Opening A Pull Request

1. Check whether an issue already exists.
2. Keep the change focused.
3. Add or update tests when the change affects behavior.
4. Update documentation when behavior, setup, or usage changes.
5. Make sure no secrets, private data, generated credentials, or local-only files are included.

## Pull Request Expectations

A good pull request should include:

- A clear summary of the change.
- The reason the change is needed.
- How it was tested.
- Any limitations or follow-up work.

## Maintainer Review

Maintainers may ask for changes before merging. Reviews should focus on correctness, maintainability, security, and project scope.

## Code Of Conduct

Contributors are expected to be respectful and constructive. If a formal code of conduct is added later, it should be linked from this section.
