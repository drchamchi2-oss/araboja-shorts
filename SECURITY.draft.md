# Security Policy Draft

This file is a draft. Review and replace contact details before making the repository public.

## Supported Versions

Until the project has public releases, only the default branch is considered for security review.

| Version | Supported |
| ------- | --------- |
| main    | Yes       |

## Reporting A Vulnerability

Please do not open a public issue for suspected security vulnerabilities.

Instead, contact the maintainer privately through the preferred channel listed below:

```text
Security contact: [add email or GitHub Security Advisory process]
```

A useful report should include:

- A short description of the issue.
- Steps to reproduce.
- Impact and affected versions or commits.
- Any suggested fix or mitigation.

## Scope

Only report vulnerabilities in code that is part of this repository and that you are authorized to test. Do not scan, probe, or test third-party systems without permission.

## Maintainer Response

The maintainer will review the report, ask for additional information if needed, and coordinate a fix or disclosure timeline where appropriate.
